# Gemini-Proxy Tools Schema Format Specification

**Date:** 2026-07-11  
**Status:** Approved  
**Author:** Claude Code (Fable 5)  

---

## 1. Overview
This design specification defines the corrections for the `tools` parameters mapping and schema translation inside **gemini-proxy**. During tool testing, raw parameters generated by Claude clients contained draft metadata or properties that Gemini's strict REST validator rejected with 400 validation errors. This specification resolves those issues by introducing recursive schema sanitization mirroring the proven logic in `GeminiToAPI`.

---

## 2. Root Cause Analysis
Gemini's REST JSON Schema validation is extremely strict and rejects requests containing any of the following parameters or schemas:
1. **Unsupported Draft Metadata Keywords:** Blacklisted properties like `$schema`, `additionalProperties`, `$ref`, `ref`, `const`, `default`, `examples`, etc., cause immediate validation failures.
2. **Lower-Case Type Values:** Claude and standard schemas specify types in lowercase (e.g., `{"type": "string"}`). Gemini strictly requires uppercase type strings (e.g., `{"type": "STRING"}`).
3. **Type Array Mapping:** Standard schemas often use arrays for types (e.g., `["string", "null"]` to denote nullable values). Gemini's REST schema does not support arrays of types; it requires setting `nullable: true` along with a single uppercase string `type` property.

---

## 3. Corrected Mapping Design

### 3.1. Recursive Schema Sanitizer (`_convertSchemaToGemini`)
We will implement `_convertSchemaToGemini` in `src/services/claudeTranslator.js` to recursively process tool schemas:
- Clean and strip blacklisted keywords (`$schema`, `additionalProperties`, etc.).
- Recursively cast type values to uppercase (e.g. `object` -> `OBJECT`).
- Translate nullable type arrays (e.g., `["string", "null"]` -> `type: "STRING", nullable: true`).
- Map nested schemas safely using properties-aware flag contexts (`isProperties`).

### 3.2. Tools Pipeline Integration
In the `translateClaudeToGoogle` function of `src/services/claudeTranslator.js`, update the tools mapping loop to run every tool's `input_schema` through `this._convertSchemaToGemini(tool.input_schema)`.
